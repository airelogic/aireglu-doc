Adapter Implementation

Adapter Implementation ​

Introduction ​

Adapters are an extensibility point within the AireGlu framework. They allow you to implement custom task behaviours as a code plugin, which can then be called upon within any endpoint pipeline.

Development ​

  1. Create a new dotnet core class library
    dotnet new classlib --framework netcoreapp3.1
  2. Install the AireGlu SDK
    dotnet add package AireGlu.SDK
  3. Implement IAdapter
csharp
using AireGlu.Public.DTO.ResolvedValues;
using AireGlu.SDK;

namespace Adapters
{
    [Adapter("Test")]
    public class TestAdapter : Adapter<ResolvedValues>
    {
        public override Task<TaskResult> Execute(
            ResolvedValues values,
            IEnumerable<TaskResult> taskResults,
            string transactionId) =>
            Task.FromResult(new TaskResult
            {
                Result = "Test result"
            });
    }
}

Enhancements ​

Adding properties ​

To pass context properties to your adapter, you will create a class that derives ResolvedValues

csharp
using AireGlu.Public.DTO;
using AireGlu.Public.DTO.ResolvedValues;
using AireGlu.SDK;

namespace Adapters
{
    public class MagicResolvedValues : ResolvedValues
    {
        [AdapterField(AdapterFieldEditorType.Text, Required=true)]
        public string SpellName { get; set; }

        [AdapterField(AdapterFieldEditorType.RichText)]
        public string SpellDescription { get; set; }

        public override string ToContextJson() => JsonSerializer.Serialize(this);
    }
}

This will generate UI for each custom property: You will notice that we can decorate the properties with [AdapterField] attributes, which allows us to specify metadata such as:

  • Editor type
  • Required status
  • Friendly name
  • Primary status

Editor types ​

  • Text
  • Rich text
  • Code
  • Select (a dropdown menu of preset options, defined in the Values attribute)

This attribute is not required however, and if omitted the property will take the default 'Text' editor type, with the name taken from the property, and required being false.

Improving performance ​

You can improve the performance of your adapters by using compiled serialization for your ToContextJson method, which will remove the requirement for the framework to use reflection to inspect your ResolvedValues. Firstly you need a SerializationContext:

csharp

[JsonSerializable(typeof(MagicResolvedValues))]
public partial class MagicResolvedValuesSerializerContext : JsonSerializerContext
{}

which you can then use from the ToContextJson method:

csharp
public override string ToContextJson()
  => JsonSerializer.Serialize(this, MagicResolvedValuesSerializerContext.Default.MagicResolvedValues);

Adapter dependencies ​

Your adapter implementation can have dependencies injected from AireGlu, using the [ImportingConstructor] attribute:

csharp
using System.Collections.Generic;
using System.Net.Http;
using System.Text;
using System.Threading.Tasks;
using System.Composition;

using AireGlu.SDK;

namespace Adapters
{
    [Adapter("Slack")]
    public class SlackAdapter : Adapter<SlackResolvedValues>
    {
        private HttpClient _client;

        [ImportingConstructor]
        public SlackAdapter(IHttpClientFactory httpClientFactory)
        {
            _client = httpClientFactory.CreateClient();
        }

        public override async Task<TaskResult> Execute(
            SlackResolvedValues values,
            IEnumerable<TaskResult> taskResults,
            string transactionId)
        {
            var result = await _client.PostAsync(
                values.WebHookUrl,
                new StringContent(
                    "{\"text\":\"hello\"}",
                    Encoding.UTF8,
                    "application/json"));

            return new TaskResult()
            {
                Success = result.IsSuccessStatusCode,
                Result = await result.Content.ReadAsStringAsync()
            };
        }
    }
}

In order to inject a dependency, it must be supported by the core framework. The current list of available imports is as follows:

  • System.Net.Http.IHttpClientFactory

Publishing ​

In order to publish your adapter and make it available within your SaaS cloud environment, it needs to be submitted for approval. This is a light-touch governance process which can be started by raising a support ticket.

Note: To access the output of an adapter via liquid

{{ endpoint.Tasks[indexOfAdapterTask] }}

This will return:

{
    "Format": "XML",
    "StatusCode": 200,
    "Result": "result of mapping-task",
    "Success": true,
    "ContentType": "application/xml",
    "RedirectUrl": null,
}
