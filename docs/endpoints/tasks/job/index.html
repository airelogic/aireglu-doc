    Job Task

    Mar 29, 2021

    The Job task is much like it’s cousin, the Jobs page, but has the ability to create and remove jobs from within an Endpoint instead of manually. This allows you to automatically create or delete jobs as needed.

    Creating a job


    To create a job, ensure Create is the selected option. From here you can name the job, select the endpoint you want to run, what data it should have, and set the schedule. You can also set Disable on failure, to turn the job off if it doesn’t work. You can always turn it back on manually later. When defining the job name, you should try and ensure it will be unique. The name and the endpoint data can both accept fields from the context panel.


    For setting the schedule, you can either use the wizard options to set a pattern, or use a cron expression. Cron expressions are in the standard Linux format. For advanced use cases, you can define the schedule using fields from the Context panel as well.

    When you’re done, simply click Next to go to your next task.

    Deleting a job

    To remove a job is pretty simple. All you need to do is choose Remove as your option, and enter the job name. Like creating a job, this can include data fields from the Context panel, and should be the name of the job you want to delete.
